O que é o Git

▶

O Git é um sistema de controle de versão distribuído, criado por Linus Torvalds em 2005. Ele permite que desenvolvedores acompanhem mudanças no código-fonte durante o desenvolvimento de software.

Vantagens do Git:

  • Desenvolvimento não linear com branches
  • Funcionamento offline
  • Rápido e eficiente mesmo em projetos grandes
  • Histórico completo do projeto

O que é o GitHub

▶

O GitHub é uma plataforma de hospedagem de código baseada na web que utiliza o Git para controle de versão. Foi adquirido pela Microsoft em 2018.

Recursos principais:

  • Hospedagem de repositórios Git
  • Ferramentas de colaboração (issues, pull requests)
  • GitHub Pages para hospedagem de sites
  • Integração com CI/CD (GitHub Actions)
  • Controle de acesso e permissions

Diferenças entre Git e GitHub

▶
  • Git: Ferramenta de linha de comando para controle de versão local. Pode ser usado sem internet.
  • GitHub: Serviço online que hospeda repositórios Git, adicionando funcionalidades sociais e colaborativas.
  • Git é software livre e gratuito; GitHub oferece planos gratuitos e pagos com recursos extras.
  • Git foca no versionamento; GitHub foca na colaboração e hospedagem.
  • Alternativas ao GitHub: GitLab, Bitbucket, SourceForge.

Comandos Linux no Git Bash

▶

O Git Bash é um emulador de terminal para Windows que permite usar comandos Unix. Comandos essenciais:

  • ls: Listar arquivos e diretórios list
  • cd [diretório]: Mudar de diretório change directory
  • pwd: Mostrar diretório atual print working directory
  • mkdir [nome]: Criar diretório make directory
  • rm -r [nome]: Remover diretório e conteúdo remove recursive
  • mv [origem] [destino]: Mover/renomear move
  • cp [origem] [destino]: Copiar arquivos copy
  • touch [arquivo]: Criar arquivo vazio

Comandos Git Essenciais

▶

Comandos básicos para começar com Git:

  • git init: Inicializar repositório
  • git clone [url]: Clonar repositório existente
  • git add [arquivo]: Adicionar arquivo ao staging
  • git commit -m "mensagem": Criar commit
  • git status: Verificar estado dos arquivos
  • git push: Enviar commits para repositório remoto
  • git pull: Atualizar repositório local
  • git branch: Listar branches
Dica: Use git status frequentemente para verificar o estado do seu repositório.

Criar Repositório no GitHub

▶
  1. Acesse github.com e faça login
  2. Clique no botão "+" no canto superior direito → "New repository"
  3. Preencha o nome do repositório (ex: meu-projeto)
  4. Adicione uma descrição (opcional)
  5. Escolha entre público (visível a todos) ou privado (apenas você)
  6. Marque "Initialize with a README" para criar um arquivo README.md inicial
  7. Clique em "Create repository"
Dica: O README.md é importante para documentar seu projeto! Inclua informações sobre o que o projeto faz, como instalá-lo e como usá-lo.

Conectar Projeto Local ao GitHub

▶

Para conectar um projeto existente a um repositório GitHub:

  1. No diretório do projeto: git init
  2. Adicione os arquivos: git add .
  3. Commit inicial: git commit -m "Initial commit"
  4. No GitHub, copie a URL do repositório (HTTPS ou SSH)
  5. Adicione o remote: git remote add origin [URL]
  6. Envie: git push -u origin main
Nota: Antes era comum usar "master", mas agora o padrão é "main". Se encontrar problemas, verifique qual o nome da branch padrão do seu repositório.

Versionamento com Git

▶

Fluxo básico para criar versões (commits):

  1. Verifique mudanças: git status
  2. Adicione arquivos modificados: git add [arquivo] ou git add . (todos)
  3. Crie um commit: git commit -m "Mensagem descritiva"
  4. Verifique o histórico: git log

Boas práticas:

  • Faça commits pequenos e frequentes
  • Use mensagens de commit claras e descritivas
  • Um commit deve representar uma mudança lógica única

Enviar para o GitHub

▶

Após fazer commits locais, envie para o GitHub:

  1. Certifique-se de ter um remote: git remote -v
  2. Envie para a branch principal: git push origin main
  3. Se for a primeira vez: git push -u origin main
Dica: O -u define o upstream, então da próxima vez basta usar git push.

Para enviar uma nova branch: git push -u origin [nome-da-branch]
